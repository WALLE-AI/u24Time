# 多智能体框架深度分析报告：BettaFish, MiroFish与openclaw

本文档对开源目录下的三个智能体框架（`BettaFish`、`MiroFish`、`openclaw`）进行了深度解析，重点解读其在**智能体的技术架构**、**上下文管理**、**记忆管理**以及**工具调用**四个核心层面的设计与实现。

---

## 1. BettaFish

BettaFish是一个面向深度研究与报告生成的节点式智能体框架。在实现上更偏向于阶段性的管道（Pipeline）架构。

### 1.1 智能体技术架构
- **节点化流水线（Node-based Pipeline）**：核心类是`DeepSearchAgent`，它将整个研究过程拆解为了不同的节点。例如，它包含了`FirstSearchNode`（初步探索节点）、`ReflectionNode`（深度反思节点）与`ReportFormattingNode`（报告格式化节点）。
- **流程驱动（Workflow Driven）**：系统首先规划出整体报告的大纲，随后将其拆分成各个段落（Paragraphs），并分别为每个段落安排迭代式的研究与撰写，最后合并生成完整报告。这是一种典型的大任务拆解分配机制。

### 1.2 上下文管理
- **基于数据类（Dataclasses）的严谨状态追踪**：在`InsightEngine/state/state.py`中，系统定义了完整的State机制。
- **层次化上下文**：整体报告有总的上下文`State`；对于分段研究的独立闭环，存在`Research`类状态对象和`Paragraph`对象。通过在数据类中存储已收集的事实（Facts）、现存大纲（Outline）等变量来维护全局状态，不会混淆不同段落的背景信息。

### 1.3 记忆管理
- **短期工作记忆**：主要体现为`Research`对象内部保留的`search_history`（历史搜索路径）与中间阶段生成的`summary`。
- **阶段性结晶**：通过不断进行“总结-反思-再搜索”的信息消化，把每一次的原始参考资料转化为有核心信息提炼的短摘要（Summary），以此作为下一轮调用的记忆提示，避免了上下文长度溢出。

### 1.4 工具调用
- **封装与解析**：在节点内部（如`FirstSearchNode`），系统先向LLM输入预设的Prompt，要求其以JSON格式返回意图。
- **内置搜索集成**：由`DeepSearchAgent`中的工具调用执行器直接对接搜索引擎或网络爬虫（通过`MediaCrawlerDB`和相关的查询实现类）发起网络获取工作，解析响应后回填给状态管理器。

## 2. MiroFish (OASIS Simulation Runner)

MiroFish项目实现了一个面向图谱支持与实时推演干预的仿真智能体架构。它有着前后端分离的系统，主推演逻辑放置在后端。

### 2.1 智能体技术架构
- **双模并行架构**：包含后台的批量模拟器引擎（`SimulationRunner`）与前台基于大模型的洞察服务（`ReportAgent`）。 
- **ReAct循环执行器**：以`ReportAgent`为例，采用了典型的基于观察-思考-执行（ReAct）的架构模式，系统以迭代轮次推进。它模拟在一个类似于“上帝视角”的体系中，按不同平台（如Reddit、Twitter）驱动多个群体智能体模拟运作。

### 2.2 上下文管理
- **JSONL结构化轨迹**：智能体的动作轨迹与观察结果统一记录为强类型事件，按不同平台切分（`twitter/actions.jsonl`）。
- **模拟器状态追踪**：`SimulationRunState`维护着当前进行的轮次数、被模拟的时间轴（Simulated Hours）、当前并行平台的运行状态等等。LLM推演时，从这些上下文日志以及全局世界大纲设定去合成当前的决策背景。

### 2.3 记忆管理
- **支持Zep图数据记忆（Zep Graph Memory）**：这是框架中一大亮点核心。通过`ZepGraphMemoryManager`，它能够动态接收事件并将Agent的动态轨迹持久化为底层的知识图谱（Knowledge Graph）。
- **记忆的持久与关联**：相较于常规线性历史缓冲，这类设计有利于后期让洞察智能体发起“实体查询”、“关联分析”这种跨时间的记忆召回逻辑。

### 2.4 工具调用
- **基于`<tool_call>`标签的函数路由**：在`ReportAgent`内，模型被训练为输出带有XML语法标签的特定控制符串，而后被解析器截取后路由。
- **深层图谱挖掘服务封装 (`zep_tools.py`)**：围绕图谱记忆集成了多个专精的高级工具。例如：
  - `insight_forge`：能对宽泛查询进行层级下钻（Sub-questions breakdown）与维度检索。
  - `panorama_search`：全景数据搜索。
  - `quick_search` / `interview_agents`：让报告智能体能够模拟采访其他参与模拟环境的子Agent。

## 3. openclaw

openclaw 是一个庞大且工程化完备的TypeScript库系统。它侧重于提供安全、强大并可被各路渠道（Channel）集成的智能体核心运行时环境，其规模接近企业级的框架生态（支持多个模型底层切换，如Bedrock, Vercel, BytePlus等）。

### 3.1 智能体技术架构
- **基于Subagent与宿主运行时**：采用了更为先进的代理注册和层次派生（Spawn）模型。系统包含针对不同环境抽象的`Runner`类（如 `pi-embedded-runner`用于嵌入的内部推演环境）。通过 `Subagent-registry`管理多级智能体之间的通讯及生命周期。
- **跨渠道（Multi-channel）兼容**：架构被解耦成了能力提供（Providers）层与渠道接入（Channels）层，让智能体不仅能独立运行，还能桥接在Discord、Slack等众多环境。

### 3.2 上下文管理
- **沙盒化的安全隔离界限 (`sandbox.ts`)**：与前面框架不同，openclaw针对执行代码/控制工作区的指令具有极高的边界控制能力。上下文引擎`context-engine`包含了详尽的会话边界定义。
- **基于流(Stream)的协议层订阅交互**：在`pi-embedded-subscribe.ts`内，通过事件流来管理代理与用户的上下文刷新与分段响应。

### 3.3 记忆管理
- **矢量化后备存储引擎 (`search-manager`, `embedding-manager`)**：在`src/memory`中可以看出，其记忆管理更偏向基于语义检索的大规模历史归档机制。其不仅通过`sqlite-vec.ts`进行了轻量数据库映射，支持同步与重组(`qmd-manager`, `compaction.ts`)，还有基于时间衰减（`temporal-decay`）这种接近真实思维的知识修剪策略。

### 3.4 工具调用
- **声明式的策略与编排管道 (`tool-policy-pipeline.ts`)**：极精细的工具调度管理。针对不同的指令提供了各种约束策略，包含工作区目录安全保护(`workspace-only-false.test.ts`)和进程控制隔离。
- **大量标准宿主工具及插件扩展 (`pi-tools.ts`, `bash-tools`)**：其具备直接与Host机器进行的底层能力互动，例如强大的系统Bash执行`exec`与文件级的Patch应用能力。为了保证稳定性它具备防止LLM工具调用死循环的`tool-loop-detection`，同时还带有复杂的`model-selection`以便工具适配当前最合适且没熔断的模型后盾。

## 4. 舆情分析、事件分析与预测分析专项对比

针对框架在**舆情感知**、**事件拆解**与**未来预测**这三个核心业务维度的表现，我们重点对比了相关系统的处理方式：

### 4.1 舆情分析 (Opinion / Sentiment Analysis)
- **BettaFish**：定位为一个纯正的高级“舆情分析系统”。它内置了强大的情感分析集成模块（`sentiment_analyzer.py`），且官方包含了多个专门针对社交媒体微调的情感模型库（如基于BERT、GPT-2、Qwen的微博情感分类器）。它直接对大规模社媒评论进行正向/中性/负向情感的分类，并能生成随时间变化的舆情情绪曲线。
- **MiroFish**：MiroFish 的舆情处理不是“静态统计”，而是“动态生成”。它在构建模拟环境时（`Step2EnvSetup`），允许用户为每个 Agent 赋予“情感偏置（`sentiment_bias`，范围在 -1.0 到 1.0）”参数，使得智能体在参与讨论初期就自带立场（正负面情绪）。

### 4.2 事件分析 (Event Analysis)
- **BettaFish**：长于对**既定事实的复盘提取**。它能够从爬取的碎片化事件数据中，利用节点架构自动提取出**事件的时间轴（Timeline）**、**核心引爆点**、以及不同群体（如产业视角、投资者声音、媒体视角）维度的焦点碰撞与利益诉求。
- **MiroFish**：事件在 MiroFish 中是模拟引擎启动的“燃料”（称为`初始激活事件 / initial_posts`）。它的事件分析关注的是“进行时”，通过给群体植入事件种子和叙事基调（Narrative Direction），观察数字智能体在其设定的 MBTI、年龄、身份框架下如何围绕该事件自主做出连锁发帖与互动反应。

### 4.3 预测分析 (Predictive Analysis)
- **BettaFish**：预测手段侧重于基于现有数据分析和领域常识的**静态推理拓扑**。例如在其分析报告中，常通过分析当前趋势来直接让 LLM 总结“未来展望”，或者配合固定的算法（如蒙特卡洛仿真量化风险）来预估未来的数据波动点。
- **MiroFish**：**预测是 MiroFish 框架的核心灵魂（“预测万物”）**。它打破了单线推理，通过在“沙盘”（数字世界）中投入成百上千个 Agent，利用图数据驱动记忆交互，使得预测结果不是大模型单次生成的结论，而是成百上千次“行动-反应”迭代产生的“群体涌现”现象。\n\n*特别提示：在实战业务中，这两种系统具有极强的**互补性**（例如可以先用 BettaFish 收集真实社媒数据并梳理完现有事态时间线，随后将这份详尽的分析报告投入 MiroFish 作为“种子背景”，从而极为精准地推演未来的舆情风暴走向）。*

---

## 总结

- **BettaFish** 强在**多节点数据链路**和内置的**专属情感分类小模型池**，是生成确定性、长篇专业级舆情及事件复盘报告的利器。
- **MiroFish** 跳出了传统静态分析，采用将仿真模拟器（Simulation）与大模型 ReAct 流程结合的架构，并将个体知识抽取成 **Zep 动态图谱记忆**。其利用带有先天“情感与人设”的群体 Agent 来做“沙盘预演式”的未来预测，具有极强的可看性与动态交互性。
- **openclaw** 则是重度工程化、高扩展的全能底层。提供复杂的沙盒边界安全机制、极其完善的外挂式语义向量搜索（包含 Temporal Decay 时间衰减遗忘）以及安全宿主工具回调链。适合将其作为未来承载前两者（舆情与推演）的底层多渠道调度基座。
